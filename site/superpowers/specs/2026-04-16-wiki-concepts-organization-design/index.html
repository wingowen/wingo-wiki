    Wiki 概念页面组织设计方案

    问题分析

    当前 https://wingo-wiki.netlify.app/concepts/ 页面存在以下问题：

    1. 内容杂乱：概念文件缺乏清晰的分类体系
    2. 重复内容：存在多个重复或相似的概念文件
    3. 标题混乱：混合使用中文和英文标题
    4. 结构不清晰：缺乏层级组织和导航体系

    解决方案

    1. 分类体系设计

    采用领域 + 主题的双重分类体系：

    领域分类（一级分类）

    • Agent 架构：Agent 架构设计、框架理论与实践
    • 工具使用：Tool Use、Advanced Tool Use、Think 工具
    • 上下文工程：Context Engineering、Context Management
    • RAG 技术：RAG、HyDE、Contextual Retrieval
    • 多智能体：Multi-Agent、Multi-Agent Research
    • MCP 技术：MCP、MCP Code Execution、Desktop Extensions
    • Claude 相关：Claude Code、Claude Agent SDK
    • 其他技术：LangGraph、SWE-bench 等

    主题分类（二级分类）

    • 核心概念：基础理论和核心原理
    • 实践指南：实际应用和最佳实践
    • 面试专题：面试常见问题和解答
    • 技术深度：深入技术细节和实现

    2. 目录结构优化

    遵循 llm-wiki 的标准结构：

    wiki/
├── index.md               # 自动生成的 wiki 索引
├── log.md                 # 操作历史记录
├── concepts/              # 概念页面目录
│   ├── agent-architecture/      # 领域分类
│   │   ├── core/                # 主题分类
│   │   │   ├── agent-architecture.md    # 核心概念
│   │   │   └── agent-framework-theory.md
│   │   ├── practice/            # 主题分类
│   │   │   └── agent-framework-practice.md
│   │   └── interview/           # 主题分类
│   │       └── interview-agent-arch.md
│   ├── tool-use/
│   │   ├── core/
│   │   │   ├── tool-use.md
│   │   │   └── advanced-tool-use.md
│   │   └── practice/
│   │       ├── writing-effective-tools.md
│   │       └── think-tool.md
│   └── ...
├── sources/              # 来源页面
└── answers/              # 保存的查询答案

    3. 内容整合策略

    重复内容处理

    • 合并重复文件：将内容相似的文件合并为一个完整的文件
    • 保留核心内容：保留最完整、最新的内容
    • 创建交叉引用：在相关概念之间建立链接

    标题统一规范

    • 双语标题格式：中文标题 | English Title
    • 统一文件命名：使用英文文件名，保持一致性
    • 标准化元数据：统一 YAML frontmatter 格式

    4. 导航与索引优化

    概念索引页面

    • 按领域分组：在索引页面按领域分组显示概念
    • 主题标签：为每个概念添加主题标签
    • 搜索功能：添加概念搜索功能
    • 最近更新：显示最近更新的概念

    内部链接优化

    • 标准化链接格式：使用 [[概念名称]] 格式
    • 自动生成反向链接：在每个概念页面显示引用该概念的其他页面
    • 相关概念推荐：在页面底部显示相关概念

    5. 实施步骤

    1. 初始化 llm-wiki：在 wingo-wiki 目录中初始化 llm-wiki 结构
    2. 内容审计：识别重复内容和分类不当的概念
    3. 分类映射：将现有概念映射到新的分类体系
    4. 内容合并：合并重复内容，保留核心信息
    5. 目录重构：创建符合 llm-wiki 标准的目录结构
    6. 标题统一：更新所有概念标题为双语格式
    7. 链接更新：更新内部链接和引用为 Obsidian 风格
    8. 配置设置：配置 .wikirc.yaml 文件
    9. 索引生成：使用 llm-wiki 工具生成索引页面
    10. 测试验证：验证所有链接和导航功能

    预期效果

    • 结构清晰：概念按领域和主题有序组织
    • 内容一致：消除重复内容，保持信息一致性
    • 导航便捷：提供清晰的分类导航和搜索功能
    • 国际化：双语标题支持不同语言用户
    • 可维护性：规范的结构便于后续维护和扩展

    技术实现

    • 基于 llm-wiki：使用 Node.js 版 llm-wiki CLI 工具
    • 目录结构：遵循 llm-wiki 标准结构，按领域和主题组织文件
    • 元数据管理：使用 YAML frontmatter 管理概念元数据
    • 链接处理：使用 Obsidian 风格的内部链接 [[概念名称]]
    • 索引生成：利用 llm-wiki 的自动索引生成功能
    • 配置管理：使用 .wikirc.yaml 配置文件管理 LLM 设置

    后续维护

    • 定期审计：定期检查和整理概念内容
    • 分类更新：根据技术发展更新分类体系
    • 内容质量：确保概念内容的准确性和完整性
    • 用户反馈：收集用户反馈，持续改进组织方式
