    LLM Wiki Skill 维护规范

    1. 维护目标

    • 保持技能内容的准确性和时效性：确保技能内容与最新的 LLM 技术和最佳实践保持同步
    • 提高技能的可维护性：建立清晰的结构和流程，便于后续更新和扩展
    • 确保技能的安全性：定期审查技能内容，避免安全漏洞和潜在风险
    • 提升用户体验：提供清晰、易用的技能内容和使用指南

    2. 技能结构规范

    2.1 三层结构

    按照 Anthropic Agent Skills 的三层结构组织内容：

    1. 元数据层：
    2. 在 YAML frontmatter 中定义技能的基本信息
    3. 包括：名称、描述、版本、创建时间、更新时间、标签等

    4. 示例： 

      ---
title: "示例技能"
created: 2026-04-16
updated: 2026-04-16
version: 1.0.0
type: skill
tags: [example, llm]
---

    5. SKILL.md 层：

    6. 每个技能应包含一个 SKILL.md 文件
    7. 包含技能的核心指令、知识和使用指南

    8. 采用 Markdown 格式，结构清晰，易于阅读

    9. 附加文件层：

    10. 组织相关资源、脚本和示例
    11. 包括：
      • 脚本文件（.py、.sh 等）
      • 配置文件（.yaml、.json 等）
      • 示例代码和模板
      • 相关文档

    2.2 目录结构

    技能内容应按照以下目录结构组织：

    docs/
├── skills/                 # 技能根目录
│   ├── skill-name/         # 技能名称目录
│   │   ├── SKILL.md        # 技能核心文件
│   │   ├── scripts/        # 技能相关脚本
│   │   ├── examples/       # 示例代码和模板
│   │   └── resources/      # 相关资源文件
│   └── index.md            # 技能索引页面
└── llm-wiki-skill-maintenance.md  # 维护规范文档

    3. 内容更新流程

    3.1 更新频率

    • 核心技能：每月审查一次，确保内容准确性和时效性
    • 非核心技能：每季度审查一次
    • 安全更新：发现安全问题时立即更新

    3.2 更新步骤

    1. 内容审查：
    2. 检查内容是否准确，是否需要更新
    3. 验证链接是否有效

    4. 检查代码示例是否可执行

    5. 内容更新：

    6. 修改需要更新的内容
    7. 更新 frontmatter 中的更新时间

    8. 版本号递增（遵循语义化版本规范）

    9. 测试验证：

    10. 测试技能功能是否正常
    11. 验证内容是否符合规范

    12. 检查是否存在安全问题

    13. 部署发布：

    14. 提交更新到版本控制系统
    15. 触发自动部署流程
    16. 验证部署结果

    4. 质量控制标准

    4.1 内容质量

    • 准确性：内容必须准确反映 LLM 技术和最佳实践
    • 完整性：内容应覆盖技能的所有关键方面
    • 清晰度：内容应结构清晰，易于理解
    • 一致性：使用一致的术语和格式

    4.2 技术质量

    • 代码质量：脚本和示例代码应遵循最佳实践
    • 安全性：避免安全漏洞和潜在风险
    • 性能：确保技能执行效率高
    • 可靠性：技能应稳定运行，避免崩溃和错误

    4.3 文档质量

    • 格式规范：遵循 Markdown 格式规范
    • 链接有效：所有链接应指向有效资源
    • 版本控制：维护清晰的版本历史
    • 变更记录：记录重要的变更内容

    5. 安全注意事项

    5.1 内容安全

    • 避免敏感信息：不要在技能内容中包含敏感信息
    • 审查外部链接：确保外部链接指向可信资源
    • 防止提示注入：设计内容时考虑防止提示注入攻击

    5.2 代码安全

    • 脚本审查：审查所有脚本代码，确保安全
    • 权限控制：脚本应只执行必要的操作，避免过度权限
    • 输入验证：对所有用户输入进行验证
    • 错误处理：妥善处理错误，避免泄露敏感信息

    5.3 部署安全

    • 环境隔离：在隔离的环境中测试技能
    • 权限管理：严格控制部署和更新权限
    • 审计日志：记录所有部署和更新操作
    • 漏洞扫描：定期进行安全漏洞扫描

    6. 开发和维护流程

    6.1 开发流程

    1. 需求分析：
    2. 确定技能的目标和功能
    3. 分析用户需求和使用场景

    4. 制定技能开发计划

    5. 设计和规划：

    6. 设计技能的结构和内容
    7. 规划目录结构和文件组织

    8. 确定技术栈和工具

    9. 内容编写：

    10. 编写技能核心内容
    11. 创建必要的脚本和示例

    12. 编写使用指南和文档

    13. 测试和验证：

    14. 测试技能功能
    15. 验证内容准确性

    16. 检查安全问题

    17. 部署和发布：

    18. 提交代码到版本控制系统
    19. 部署技能到生产环境
    20. 监控技能运行状态

    6.2 维护流程

    1. 定期审查：
    2. 按照更新频率审查技能内容
    3. 检查是否需要更新

    4. 收集用户反馈

    5. 内容更新：

    6. 根据审查结果和用户反馈更新内容
    7. 修复发现的问题

    8. 优化技能性能

    9. 安全更新：

    10. 及时修复安全漏洞
    11. 更新依赖项

    12. 加强安全措施

    13. 性能优化：

    14. 分析技能执行性能
    15. 优化代码和内容

    16. 提高技能响应速度

    17. 用户反馈整合：

    18. 收集和分析用户反馈
    19. 优先解决用户关注的问题
    20. 持续改进技能体验

    • 文档生成：MkDocs

    7.2 资源管理

    • 内容模板：标准化的技能内容模板

    • 最佳实践：LLM 技能开发最佳实践
    • 参考文档：相关技术文档和资源
