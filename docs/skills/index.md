## 技能使用指南

### 如何使用技能

1. **浏览技能**：在技能目录中找到您需要的技能
2. **查看详情**：点击技能名称查看详细信息
3. **了解功能**：阅读技能的核心功能和使用方法
4. **调用技能**：在对话中提及技能名称或相关关键词
5. **提供参数**：根据技能要求提供必要的参数
6. **执行操作**：技能执行相应的操作并返回结果
7. **反馈结果**：提供反馈，帮助改进技能

### 技能开发指南

如果您想开发新技能，请参考以下资源：

- [LLM Wiki Skill 维护规范](file:///workspace/docs/llm-wiki-skill-maintenance.md)
- [Agent Skills 概念](file:///workspace/docs/concepts/other/practice/agent-skills.md)

### 技能维护指南

- **更新频率**：核心技能每月审查一次，非核心技能每季度审查一次
- **更新流程**：内容审查 → 内容更新 → 测试验证 → 部署发布
- **质量标准**：内容准确、完整、清晰、一致
- **安全注意**：避免敏感信息，审查外部链接，防止提示注入

## 贡献指南

如果您想贡献新技能或改进现有技能，请按照以下步骤操作：

1. **Fork 仓库**：在 GitHub 上 fork 项目仓库
2. **创建技能**：按照规范创建新技能或改进现有技能
3. **测试验证**：测试技能功能，确保符合规范
4. **提交 PR**：提交 pull request，描述您的更改
5. **审查合并**：项目维护者审查并合并您的更改
