# 层级导航设计

## 1. 项目背景

当前的 Wingo Wiki 文档系统使用 MkDocs 构建，采用了层级结构组织内容。用户希望实现点击标题就能导航到下一级页面的功能，形成一个完整的层级导航链，直到到达真实的内容页面。

## 2. 需求分析

### 2.1 现状

- 概念页面 (`/workspace/docs/concepts/index.md`) 包含顶级分类的链接，如 Agent Architecture、Context Engineering 等
- 每个顶级分类都有一个 `index.md` 文件，如 `/workspace/docs/concepts/agent-architecture/index.md`
- 每个顶级分类下有子分类，如 `core`、`practice`、`interview`，每个子分类都有一个 `_index.md` 文件
- 子分类下有具体的内容文件，如 `/workspace/docs/concepts/agent-architecture/core/agent-architecture.md`

### 2.2 需求

- 点击顶级分类标题（如 Agent Architecture）导航到对应的顶级分类页面
- 点击顶级分类页面标题（如 代理架构）导航到该分类下的第一个子分类页面
- 点击子分类页面标题（如 核心概念）导航到该子分类下的第一个内容页面
- 形成完整的导航链：概念页面 → 顶级分类页面 → 子分类页面 → 内容页面

## 3. 设计方案

### 3.1 方案一：修改所有标题为链接

- **实现方式**：修改所有 `index.md` 和 `_index.md` 文件，将标题修改为指向子页面的链接
- **优点**：实现简单，直接满足用户需求
- **缺点**：需要修改大量文件，维护成本较高

### 3.2 方案二：使用 MkDocs 插件

- **实现方式**：使用 MkDocs 插件或主题功能，自动为标题添加链接
- **优点**：无需修改大量文件，维护成本低
- **缺点**：可能需要额外的插件配置，功能可能受限

### 3.3 方案三：混合方案

- **实现方式**：修改关键页面的标题为链接，同时使用 MkDocs 配置优化导航
- **优点**：平衡了实现复杂度和维护成本
- **缺点**：需要一定的配置和修改

## 4. 推荐方案

**推荐方案一**：修改所有标题为链接

**推荐理由**：
- 直接满足用户需求，实现效果明确
- 不需要额外的插件或配置
- 符合 MkDocs 的文档结构设计理念
- 实现简单，易于理解和维护

**关键修改点**：
1. 修改顶级分类的 `index.md` 文件，将标题修改为指向第一个子分类的链接
2. 修改子分类的 `_index.md` 文件，将标题修改为指向第一个内容页面的链接
3. 保持内容页面的标题不变，因为它们已经是最终的内容页面

## 5. 实施计划

### 5.1 步骤

1. **准备工作**：确认所有顶级分类和子分类的结构
2. **修改顶级分类页面**：为每个顶级分类的 `index.md` 文件添加指向第一个子分类的链接
3. **修改子分类页面**：为每个子分类的 `_index.md` 文件添加指向第一个内容页面的链接
4. **测试导航**：验证整个导航链是否正常工作

### 5.2 示例修改

#### 顶级分类页面修改示例 (`/workspace/docs/concepts/agent-architecture/index.md`)

```markdown

title: "代理架构"
created: "2026-04-16"
type: section
tags: [agent, architecture]
---

# [代理架构](core/)

代理架构相关的核心概念、理论和实践指南。
```

#### 子分类页面修改示例 (`/workspace/docs/concepts/agent-architecture/core/_index.md`)

```markdown

title: "核心概念"
created: "2026-04-16"
type: section
tags: [core]
---

# [核心概念](agent-architecture.md)

代理架构的核心概念和理论基础。
```

## 6. 验证标准

- 点击概念页面的顶级分类标题（如 Agent Architecture）能导航到对应的顶级分类页面
- 点击顶级分类页面的标题（如 代理架构）能导航到该分类下的第一个子分类页面
- 点击子分类页面的标题（如 核心概念）能导航到该子分类下的第一个内容页面
- 整个导航链流畅，无 404 错误

## 7. 潜在风险

- **链接失效**：如果子分类或内容页面的结构发生变化，可能导致链接失效
- **维护成本**：如果添加新的分类或页面，需要手动更新相关的链接
- **用户体验**：标题成为链接后，可能会影响页面的视觉效果

## 8. 缓解措施

- **定期检查**：定期检查导航链接的有效性
- **标准化命名**：建立统一的文件命名和目录结构规范
- **添加注释**：在修改的文件中添加注释，说明链接的目的和指向

## 9. 结论

通过修改所有标题为链接的方式，可以实现点击标题就能导航到下一级页面的功能，形成完整的层级导航链。这种方式实现简单，直接满足用户需求，符合 MkDocs 的文档结构设计理念。

实施过程中需要注意维护链接的有效性，确保导航链的完整性和流畅性。
