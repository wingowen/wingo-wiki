## 超越权限提示：让 Claude Code 更安全更自主

> 原文链接：Beyond permission prompts: making Claude Code more secure and autonomous

> 作者：David Dworken, Oliver Weller-Davies

> 发布日期：2025-10-20

> 翻译日期：2026-04-14

---

Claude Code 的全新沙箱化（Sandboxing）功能——包括沙箱化 bash 工具和 Claude Code on the web——通过实现文件系统和网络隔离两个边界，大幅减少了权限提示，同时提升了用户安全性。

在 Claude Code 中，Claude 与你并肩编写、测试和调试代码，在你的代码库中导航、编辑多个文件，并运行命令来验证其工作成果。赋予 Claude 如此大的代码库和文件访问权限会带来风险，尤其是在遭遇提示注入（Prompt Injection）的情况下。

💡 术语解释：提示注入（Prompt Injection）是指通过恶意构造的输入内容，操纵 AI 模型执行非预期的操作。例如，攻击者可能在代码注释中嵌入隐藏指令，诱使 Claude 执行危险命令。

为解决这一问题，我们在 Claude Code 中引入了两项基于沙箱化技术的新功能，旨在为开发者提供更安全的工作环境，同时让 Claude 能够更自主地运行，并减少权限提示。在我们的内部使用中，我们发现沙箱化技术安全地将权限提示减少了 84%。通过定义 Claude 可以自由工作的边界，这些功能同时提升了安全性和自主性。

#### 在 Claude Code 中保障用户安全

Claude Code 采用基于权限的模型：默认情况下它是只读的，这意味着在进行任何修改或运行任何命令之前，它都需要请求权限。也有一些例外情况：我们会自动放行诸如 echo 或 cat 等安全命令，但大多数操作仍需要显式批准。

不断点击"批准"会拖慢开发节奏，并可能导致"审批疲劳"（Approval Fatigue）——用户可能不再仔细审查所批准的内容，反而降低了开发安全性。

为解决这一问题，我们为 Claude Code 推出了沙箱化功能。

### 沙箱化：一种更安全、更自主的方法

沙箱化（Sandboxing）创建了预定义的边界，Claude 可以在此边界内更自由地工作，而无需为每个操作单独请求权限。启用沙箱化后，权限提示大幅减少，安全性显著提升。

💡 术语解释：沙箱化（Sandboxing）是一种安全技术，将程序运行限制在隔离的环境中，防止其对系统其他部分造成影响。就像在真正的沙箱里玩耍一样，无论怎么折腾，沙子都不会溢出到外面。

我们的沙箱化方案基于操作系统级别的功能来实现两个边界：

  1. 文件系统隔离（Filesystem Isolation），确保 Claude 只能访问或修改特定目录。这在防止被提示注入的 Claude 修改敏感系统文件方面尤为重要。

  2. 网络隔离（Network Isolation），确保 Claude 只能连接到已批准的服务器。这可以防止被提示注入的 Claude 泄露敏感信息或下载恶意软件。

值得注意的是，有效的沙箱化需要同时具备文件系统隔离和网络隔离。没有网络隔离，被攻破的代理（Agent）可能会窃取 SSH 密钥等敏感文件；没有文件系统隔离，被攻破的代理可以轻松逃出沙箱并获取网络访问权限。正是通过同时使用这两种技术，我们才能为 Claude Code 用户提供更安全、更快速的智能体体验。

💡 举例说明：想象 Claude Code 是一个在办公室工作的助手。文件系统隔离就像给助手配了一把只能打开特定文件柜的钥匙，而网络隔离就像只允许助手拨打白名单上的电话号码。两者缺一不可——只有钥匙而没有电话限制，助手仍可能通过电话泄露机密；只有电话限制而没有钥匙限制，助手可以直接打开保险箱。

#### Claude Code 中的两项沙箱化新功能

我们推出了一款全新的沙箱运行时，以 Beta 研究预览版的形式提供，让你可以精确控制代理（Agent）能够访问的目录和网络主机，无需启动和管理容器的额外开销。该运行时可用于沙箱化任意进程、代理和 MCP 服务器。它也作为开源研究预览版提供。

在 Claude Code 中，我们使用该运行时对 bash 工具进行沙箱化，允许 Claude 在你设定的范围内运行命令。在安全的沙箱内，Claude 可以更自主地运行，安全地执行命令而无需权限提示。如果 Claude 尝试访问沙箱外部的内容，你会立即收到通知，并可以选择是否允许。

我们基于操作系统级别的原语（Primitives）构建了这一功能，例如 Linux bubblewrap 和 MacOS seatbelt，在操作系统层面强制执行这些限制。它们不仅覆盖 Claude Code 的直接交互，还包括命令产生的任何脚本、程序或子进程。

💡 术语解释：操作系统级原语（OS-level Primitives）是操作系统提供的底层安全机制。Linux bubblewrap 是一种轻量级容器化工具，通过 Linux 命名空间（Namespace）实现进程隔离；MacOS seatbelt 是苹果的沙箱执行框架，通过策略配置文件限制进程能力。

如上所述，该沙箱同时强制执行：

  1. 文件系统隔离，允许对当前工作目录的读写访问，但阻止修改其外部的任何文件。

  2. 网络隔离，仅允许通过连接到沙箱外部运行的代理服务器的 Unix 域套接字（Unix Domain Socket）进行互联网访问。该代理服务器对进程可连接的域名执行限制，并处理用户新请求域名的确认。如果你希望进一步提升安全性，我们还支持自定义该代理，对出站流量执行任意规则。

💡 术语解释：Unix 域套接字（Unix Domain Socket）是一种进程间通信（IPC）机制，允许同一台计算机上的进程之间进行数据交换。与网络套接字不同，它不经过网络协议栈，因此更高效且更安全。

两个组件均可配置：你可以轻松选择允许或禁止特定的文件路径或域名。

Claude Code 的沙箱化架构通过文件系统和网络控制隔离代码执行，自动放行安全操作，阻止恶意操作，仅在必要时请求权限。

沙箱化确保即使成功发生提示注入，也会被完全隔离，不会影响用户的整体安全。这样，被攻破的 Claude Code 无法窃取你的 SSH 密钥，也无法向攻击者的服务器回传数据。

要开始使用此功能，请在 Claude Code 中运行 /sandbox，并查看我们安全模型的更多技术细节。

为了让其他团队更方便地构建更安全的代理，我们已将此功能开源。我们认为其他团队也应考虑为其代理采用这项技术，以增强代理的安全态势。

今天，我们还发布了 Claude Code on the web，使用户能够在云端的隔离沙箱中运行 Claude Code。Claude Code on the web 在隔离的沙箱中执行每个 Claude Code 会话，使其能够以安全的方式完全访问其服务器。我们设计了该沙箱，确保敏感凭据（如 git 凭据或签名密钥）永远不会与 Claude Code 一起存放在沙箱内。这样，即使沙箱中运行的代码被攻破，用户也能免受进一步损害。

Claude Code on the web 使用自定义代理服务（Proxy Service）透明地处理所有 git 交互。在沙箱内部，git 客户端使用自定义的范围限定凭据（Scoped Credential）向该服务进行身份验证。代理验证此凭据和 git 交互的内容（例如确保只推送到配置的分支），然后在将请求发送到 GitHub 之前附加正确的身份验证令牌。

💡 术语解释：范围限定凭据（Scoped Credential）是一种权限受限的身份验证凭据，仅允许执行特定操作。例如，一个范围限定的 git 凭据可能只允许向特定仓库的特定分支推送代码，而无法执行其他操作。

Claude Code 的 Git 集成通过安全代理路由命令，验证身份验证令牌、分支名称和仓库目标——在允许安全的版本控制工作流的同时，防止未授权的推送。

### 开始使用

我们的全新沙箱化 bash 工具和 Claude Code on the web 为使用 Claude 进行工程开发的开发者带来了安全性和生产力方面的显著提升。

要开始使用这些工具：

  1. 在 Claude 中运行 /sandbox，并查看我们的文档了解如何配置此沙箱。

  2. 访问 claude.com/code 体验 Claude Code on the web。

或者，如果你正在构建自己的代理，请查看我们的开源沙箱化代码，并考虑将其集成到你的工作中。我们期待看到你的成果。

要了解更多关于 Claude Code on the web 的信息，请查看我们的发布公告。
