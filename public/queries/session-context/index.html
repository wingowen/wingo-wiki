Session Context 影响技能选择的问题分析与解决方案

1. 问题描述

当用户询问一个问题时，大模型返回的 reasoning_content 显示，模型尝试使用 web_search 工具来获取金蝶和用友最近一周的新动向，但由于之前的 web_search 调用失败，模型仍然继续尝试使用相同的工具，而没有考虑其他可用的技能。

2. 根本原因分析

通过对代码库的分析，发现以下根本原因：

2.1 会话历史管理问题

  • 失败记录持久化：工具调用失败的结果会被保存到会话历史中（SessionManager 类）
  • 历史影响：模型在生成回复时会参考会话历史，导致重复尝试相同的失败方法
  • 缺乏智能过滤：系统没有对失败的工具调用进行智能过滤或标记

2.2 技能提示问题

  • 技能信息不足：技能提示中没有包含工具的成功率或可用性信息
  • 缺乏替代方案：当一个工具失败时，没有明确的机制推荐替代技能
  • 静态技能列表：技能列表是静态的，没有基于历史表现进行动态调整

2.3 上下文管理问题

  • 上下文过载：会话历史可能包含过多的失败信息，导致模型关注于过去的失败
  • 缺乏上下文优先级：系统没有对上下文中的不同信息设置优先级
  • 历史压缩不足：没有对重复的失败模式进行压缩或清理

3. 解决方案

3.1 会话历史管理解决方案

3.1.1 智能历史过滤

  • 失败标记：对工具调用失败的记录添加失败标记

  • 过滤策略：在构建上下文时，过滤掉近期失败的工具调用

  • 历史优先级：基于成功率调整历史记录的优先级 3.1.2 历史压缩

  • 重复模式检测：检测并压缩重复的失败模式

  • 定期清理：定期清理长时间未使用的失败记录

  • 历史摘要：对历史记录进行摘要，保留关键信息

4. 实现建议

4.1 会话历史管理实现

  • 修改 SessionManager 类，添加失败标记和过滤功能
  • 实现历史压缩和摘要功能
  • 添加上下文重置机制

5. 测试策略

5.1 功能测试

  • 测试会话历史管理功能
  • 测试技能提示增强功能
  • 测试上下文管理优化功能
  • 测试错误处理优化功能

6. 预期效果

通过实施上述解决方案，预期可以：

  1. 提高工具选择的准确性：模型能够根据历史表现选择更合适的工具
  2. 减少重复失败：避免重复尝试已经失败的工具
  3. 增加技能多样性：鼓励模型使用不同的技能
  4. 提高系统的可靠性：通过容错处理和降级策略提高系统的可靠性
  5. 改善用户体验：减少错误和提高响应速度

7. 结论

Session context 影响技能选择的问题是一个复杂的系统问题，需要从会话历史管理、技能提示增强、上下文管理和错误处理等多个方面进行优化。通过实施上述解决方案，可以显著提高系统的性能和用户体验，使模型能够更智能地选择合适的技能来解决问题。
